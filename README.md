# 📒 쉬운 가계부 (Easy Budget)

**다중 인증 회원가입·로그인 흐름을 중심으로 설계한 React/Vite 포트폴리오 프로젝트입니다.**  
마이데이터를 연동한다는 가정하에, 서비스를 제작했습니다.
가계부 기능도 있지만 **실제 금융·플랫폼 서비스에서 복잡한 인증 UX와 사용자 상태 관리**를 프론트엔드 관점에서 어떻게 구조화하는지를 보여주는 데 목적이 있습니다.

## ✨ 한눈에 보는 프로젝트 포인트

- 이메일 / Gmail / 금융인증서 **다중 인증 플로우 구현**
- 인증 수단이 달라도 **동일 사용자로 병합 처리**
- 로그인 상태에 따른 **UI·데이터 접근 제어**
- 페이지 나열이 아닌 **서비스 흐름 중심 구조 설계**
- 실서비스 기준을 고려한 **인증 UX 분기 처리**

## 🛠 기술 스택 (Tech Stack)

### Frontend
- **React 18**
- **Vite**
- JavaScript (ES6+)

### State & Architecture
- **Context API** (AuthContext)
- 단일 App 상태 기반 화면 흐름 관리
- `sessionStorage`를 활용한 로그인 상태 유지

### UI / DX
- **Suspense** 기반 지연 로딩
- **Storybook**을 통한 컴포넌트 단위 검증
- 하단 Sheet / Modal 중심의 모바일 친화 UI 패턴

## 🧭 서비스 흐름 구조

```
MAIN
 └─ 로그인
     → INTRO
        → 로그인 (이메일 | 금융인증서)
            → 회원가입
                → WELCOME
                    → MAIN
```

- 비로그인 상태: 로그인 버튼만 노출
- 로그인 상태: 마이프로필 / 로그아웃 노출
- 인증 완료 후 메인 화면으로 자연스럽게 복귀

> 라우터 중심이 아닌 **“사용자 상태 중심 흐름”**으로 설계했습니다.

---

## 🔐 인증 및 계정 설계

### AuthContext 중심 구조
- 인증 상태, 사용자 정보, 병합 로직을 **하나의 Context로 관리**
- 외부 API가 없어도 동작하는 **독립적인 데모 구조**
- 새로고침 이후에도 로그인 유지

### 지원 인증 방식
- 이메일 / 비밀번호 로그인
- Gmail 연동 로그인
- 금융인증서 로그인 및 가입

### 사용자 병합 로직
- 기준 키
  - 이메일
  - 휴대폰번호 + 생년월일
- 인증 수단이 달라도 **하나의 사용자로 인식**
- 실제 금융 서비스의 중복 계정 문제를 프론트엔드 레벨에서 해결
- “로그인 방식”이 아닌 “사용자 식별” 관점의 구조 설계

## 🧪 데모 계정 정책

- 일부 계정만 실제 가계부 데이터 노출
- 그 외 계정 및 비로그인 상태는 모든 수치를 `??`로 마스킹
- → **로그인/연동의 의미를 UI에서 즉시 체감**하도록 설계

---

## 🤔 왜 이렇게 구현했는가

### 1️⃣ 인증 흐름이 서비스에서 가장 복잡하기 때문
실제 금융·플랫폼 서비스에서는 CRUD 화면보다 **회원가입·로그인·인증 흐름**이 훨씬 복잡합니다.
이 프로젝트는 그 복잡도를 **프론트엔드에서 어떻게 구조화할 수 있는지**를 보여주기 위해 시작했습니다.

---

### 2️⃣ “페이지”보다 “흐름”을 먼저 설계
- 라우터 분리보다 **사용자 상태에 따른 화면 전환**을 우선
- `App.jsx`에서 전체 흐름을 한눈에 파악 가능
- 인증 성공/실패에 따른 다음 행동이 코드 구조에 그대로 드러남

---

### 3️⃣ 실서비스 기준 UX 판단 반영
- 금융인증서 가입 시 단계별 진행 및 실패 UX 처리
- 비로그인 상태에서 주요 액션 시도 시 회원가입 유도

---

### 4️⃣ 가계부 기능은 의도적으로 단순화
이 프로젝트의 목적은 **인증 이후 서비스 경험을 보여주는 것**이기 때문에  
가계부 영역은 데모 수준으로 유지했습니다.

---

## 📌 정리

이 프로젝트는 단순 기능 구현이 아닌,
- 사용자 상태
- 인증 흐름
- 서비스 UX분기
를 프론트엔드에서 어떻게 설계하고 관리할 수 있는지를 보여주기 위한 포트폴리오용 데모입니다.
